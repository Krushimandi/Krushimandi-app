## New Translation Keys Added

The following keys were added to fix hardcoded text:

1. **`notifications.filters.request`** - For the "Request" notification type tag
2. **`notifications.offer.coupon`** - For "Coupon" label in promotion offers
3. **`notifications.offer.valid`** - For "Valid" label in promotion offers
4. **`notifications.offer.version`** - For "Version" label in update offers
5. **`notifications.offer.request`** - For "Request" label in request offers
6. **`notifications.offer.status`** - For "Status" label in request offers

## Implementation Steps

1. **Locate your i18n translation files** (usually in `src/locales/` or `src/i18n/`)
2. **Add the new keys** under the `notifications.offer` and `notifications.filters` sections
3. **Test all languages** to ensure translations appear correctly
4. **Verify RTL languages** if you support them (Arabic, Hebrew, etc.)

## Example Usage in Code

```tsx
// Before (hardcoded)
<Text>Coupon: {couponCode} | Valid: {validity}</Text>

// After (translated)
<Text>{t('notifications.offer.coupon')}: {couponCode} | {t('notifications.offer.valid')}: {validity}</Text>
```
