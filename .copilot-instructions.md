# Copilot Instructions – KrushiMandi (React Native + Firebase)

**Role:**  
Act as a senior frontend/mobile engineer building **KrushiMandi**, a production-grade AgriTech platform.  
Always assume this is **real-world production code** that will be used by thousands of users.

---

## ✅ General Guidelines
- **Use only latest, stable, and non-deprecated APIs** (React, React Native, Firebase).
- Write **clean, modular, maintainable code** following **SOLID**, **DRY**, and **KISS** principles.
- Prefer **functional components + hooks** (no class components).
- Use **clear, descriptive naming** for variables, functions, and files.
- Add **comments** for any non-trivial or complex logic.

---

## 🔒 Security & Best Practices
- **No hardcoded API keys** → always use `.env`.
- Follow **Firebase Security Rules** strictly.
- Validate user inputs and sanitize data before writing to Firestore.
- Never expose sensitive data in client-side code.
- Implement proper error handling (`try/catch`) for all Firebase calls.

---

## 🎯 Firebase Usage
- Use **Firestore with indexed queries** and pagination (`startAfter`, `limit`).
- Use **batch writes / transactions** when updating multiple documents.
- Enable **offline persistence** for Firestore when possible.
- Use **Firebase Auth** for authentication (phone/email/social).
- Show **helpful error messages** for auth and database errors.
- Prefer `onSnapshot` for real-time updates, but use unsubscribes to avoid memory leaks.

---

## 📱 React Native & React
- Use **TailwindCSS** for styling (no inline styles unless trivial).
- Ensure **responsive UI** and follow **accessibility (a11y)** guidelines.
- Use **FlatList/SectionList** with `keyExtractor`, `getItemLayout`, and `memoization` for large data.
- Optimize for performance (lazy-load heavy components/screens).
- Avoid prop-drilling → use **Zustand** or **Context API** for global state.
- Keep components **small and reusable**.

---

## ⚡ Performance
- Minimize unnecessary re-renders (`React.memo`, `useCallback`, `useMemo`).
- Use **code-splitting** / lazy imports where possible.
- Prefer `useEffect` cleanups and unsubscribes for listeners.

---

## 🧪 Testing & Maintainability
- Suggest **unit tests** or at least sanity checks for critical flows (Auth, Firestore queries).
- Keep folder structure clean: `/components`, `/screens`, `/hooks`, `/store`.
- Write **meaningful commit messages** and maintain consistent code style.

---

## 🎨 UI/UX
- Use **modern UI patterns**: smooth animations, skeleton loaders, proper spacing.
- Ensure UI handles **loading, error, and empty states** gracefully.
- Design for **low network speeds** (graceful fallbacks, cached data).
