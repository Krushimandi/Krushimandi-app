# KrushiMandi - Scalable React Native App Architecture

A well-structured, scalable React Native application for connecting farmers and buyers in the agriculture marketplace.

## 📁 Project Structure

```
KrushiMandi/
├── android/               # Android-specific native code (autogenerated)
├── ios/                   # iOS-specific native code (autogenerated)
├── src/
│   ├── assets/            # Images, fonts, icons
│   │   ├── images/        # App images and illustrations
│   │   ├── fonts/         # Custom fonts
│   │   └── icons/         # App icons and SVGs
│   ├── components/        # Reusable UI components (preserved existing)
│   ├── constants/         # App-wide constants
│   │   ├── Colors.ts      # Color palette for light/dark themes
│   │   ├── Typography.ts  # Font sizes, weights, and text styles
│   │   ├── Spacing.ts     # Layout dimensions and spacing
│   │   ├── AppConstants.ts # Enums, API endpoints, storage keys
│   │   └── index.ts       # Exports all constants
│   ├── navigation/        # React Navigation setup
│   │   ├── AppNavigator.tsx # Main navigation configuration
│   │   └── index.ts       # Navigation exports
│   ├── screens/           # Screen components organized by feature
│   │   ├── auth/          # Authentication screens
│   │   │   ├── LoginScreen.tsx
│   │   │   ├── RegisterScreen.tsx
│   │   │   ├── OTPVerificationScreen.tsx
│   │   │   └── ForgotPasswordScreen.tsx
│   │   ├── home/          # Home/Dashboard screens
│   │   │   └── HomeScreen.tsx
│   │   ├── splash/        # Splash screen
│   │   │   └── SplashScreen.tsx
│   │   ├── profile/       # User profile screens
│   │   ├── orders/        # Order management screens
│   │   ├── notifications/ # Notification screens
│   │   └── quality/       # Quality grading screens (future)
│   ├── services/          # API services and HTTP client
│   │   ├── httpClient.ts  # Axios configuration with interceptors
│   │   ├── authService.ts # Authentication API calls
│   │   └── index.ts       # Service exports
│   ├── store/             # Zustand state management
│   │   ├── authStore.ts   # Authentication state
│   │   ├── appStore.ts    # Global app state (theme, settings)
│   │   └── index.ts       # Store exports
│   ├── types/             # TypeScript type definitions
│   │   └── index.ts       # All app types and interfaces
│   ├── utils/             # Utility functions
│   │   ├── device.ts      # Device-specific utilities
│   │   ├── validation.ts  # Form validation functions
│   │   └── index.ts       # Utility exports
│   ├── hooks/             # Custom React hooks
│   │   └── index.ts       # Hook exports
│   └── config/            # Environment and app configuration
│       └── index.ts       # Config management
├── App.tsx                # Entry point with navigation and providers
├── index.js               # Root component registration
├── package.json           # Dependencies and scripts
├── tsconfig.json          # TypeScript configuration
├── tailwind.config.js     # NativeWind/TailwindCSS setup
└── babel.config.js        # Babel configuration
```

## 🏗️ Architecture Features

### ✅ State Management (Zustand)
- **Auth Store**: User authentication, login/logout, token management
- **App Store**: Theme, language, onboarding status, notification settings
- **Persistent Storage**: Auto-saves to AsyncStorage

### ✅ Navigation (React Navigation v6)
- **Stack Navigation**: Auth flow, splash screens
- **Tab Navigation**: Main app with bottom tabs
- **Type-safe Navigation**: Full TypeScript support

### ✅ API Service Layer
- **HTTP Client**: Axios with interceptors for auth headers
- **Error Handling**: Centralized error management
- **Token Refresh**: Automatic token renewal
- **Request/Response Logging**: Development debugging

### ✅ Theme System
- **Light/Dark Mode**: Complete theming support
- **Color Constants**: Organized color palette
- **Typography System**: Consistent text styles
- **Layout Constants**: Spacing and dimensions

### ✅ TypeScript Support
- **Type Definitions**: Complete app type coverage
- **API Types**: Request/response interfaces
- **Navigation Types**: Type-safe navigation params
- **Store Types**: Typed state management

### ✅ Performance Optimizations
- **Lazy Loading**: Screens loaded on demand
- **Image Optimization**: Fast Image/Expo Image ready
- **Code Splitting**: Modular architecture
- **Async Storage**: Efficient data persistence
